# สรุปส่วนงานที่ได้ดําเนินการ (Development Summary)

ในช่วงที่ผ่านมา เราเริ่มวางโครงสร้างระบบและมีการพัฒนาทั้งในส่วนของ **Backend (NestJS)** และ **Mobile App (Flutter)** โดยเน้นที่การเก็บข้อมูลแปลงเกษตร การคำนวณคาร์บอน และการปรับการแสดงผลให้เข้ากับการใช้งานในไทย ดังนี้:

## 1. ระบบจัดการแปลงเกษตรและระบบพิกัด (Plots)

- **โครงสร้างรับข้อมูลแปลง (DTO):** ออกแบบรูปแบบการรับข้อมูลการสร้างแปลง (Plot) ที่ผูกติดกับแต่ละฟาร์ม (`farmId`)
- **การแยกประเภทแปลงเกษตร:** รองรับการระบุประเภทแหล่งน้ำที่ใช้ เช่น แบบอาศัยน้ำฝน (Rainfed), ชลประทาน (Irrigated) หรืออื่นๆ
- **การวาดขอบเขตแปลง (Coordinates):** ระบบรองรับการบันทึกพิกัดของแปลงในรูปแบบจุดพิกัด (Latitude/Longitude) เพื่อใช้ระบุขอบเขตพื้นที่จริง

## 2. ระบบประเมินและบริหารจัดการคาร์บอนเครดิต (Carbon Calculation)

- **API การคำนวณคาร์บอนเครดิต:** สร้าง Endpoints สำหรับรับค่าตัวแปร เช่น ขนาดพื้นที่ (ไร่), ปริมาณปุ๋ยไนโตรเจนที่ใช้, การใช้เทคนิคเปียกสลับแห้ง (AWD) และระยะเวลาเพาะปลูก ដើម្បីนำไปคำนวณหาปริมาณคาร์บอน (`POST /carbon/calculate`)
- **API การบันทึกและแสดงผล:** มีระบบบันทึกผลการคำนวณที่ได้เก็บไว้ในระบบ (`POST /carbon/save`) รวมถึงมี Endpoint สำหรับดึงข้อมูลภาพรวมคาร์บอนของแต่ละโครงการ (`GET /carbon/summary/:siteId`)

## 3. ระบบรองรับการใช้งานหลายภาษาในแอปพลิเคชัน (Localization)

- **ระบบ AppLocalizations (Mobile App):** เริ่มต้นการวางรากฐานให้แอปพลิเคชันมือถือรองรับหลายภาษา (ปัจจุบันคือภาษาไทยและภาษาอังกฤษ) โดยดึงคำแปลจากไฟล์ JSON

## 4. ระบบจัดการหน่วยวัดพื้นที่เกษตรแบบไทย (Thai Area Formatting)

- **ฟังก์ชันแปลงหน่วยพื้นที่ (Mobile App):** เขียนระบบประมวลผลการคำนวณขนาดที่ดิน (เนื้อที่ไร่แบบทศนิยม) ให้แสดงผลออกมาในรูปแบบมาตรฐานไทยที่คุ้นเคย คือ **ไร่-งาน-ตารางวา**
- _ตัวอย่างการทำงาน:_ หากระบบได้รับพื้นที่ตัวเลขเศษส่วนทศนิยม ระบบจะทอนหน่วยอัตโนมัติ (1 ไร่ = 4 งาน, 1 งาน = 100 ตารางวา) เพื่อแสดงผลได้แม่นยำและอ่านง่ายที่สุด
